## NEW TASK: Replace Contact ID with Phone for Deletion

### Problem
Contact ID mismatches occur when deleting attendance records. The backend API supports deletion by phone number + date combination.

### Backend Endpoint (provided by user)
```
DELETE /attendance/?phone=+27798345993&date=2026-04-21
```

### Current Flow
1. User long-presses attendance record in history screen
2. Dialog confirms deletion
3. Provider calls `deleteAttendanceForContact(contactId: ..., serviceDate: ...)`
4. Repository calls local + remote delete with `contactId` parameter
5. Remote API uses `contact_id` query parameter

### Proposed Solution
Replace `contactId` with `phone` throughout the deletion chain.

### Files to Modify

#### 1. `lib/features/attendance/domain/repositories/attendance_repository.dart`
- Change `deleteAttendanceFiltered` signature:
  - Replace `int? contactId` with `String? phone`

### Additional Considerations

1. **Date parameter**: The existing code already uses `date` (single date) for deletion, which matches the API requirement.

2. **Phone format**: Ensure phone numbers are in international format (e.g., +27798345993). The `Attendance` model stores phone as a string. The QR scanner normalizes to +27 format. Should be consistent.

3. **Local Database Indexing**: The `phone` column in the `Attendances` table should be indexed if not already. Check database schema - if no index, consider adding for performance.

4. **Backward compatibility**: This is a breaking change for local operations. Any local pending sync operations that use old `contactId` format will need migration? 
   - **Check**: Attendance records stored for sync (in `SyncQueue` table) - they store the full attendance data as JSON. The deletion is immediate-local (not queued), so no migration needed.

5. **Other usages of deleteAttendanceFiltered**: Search for other calls to this method.
   - Only used in `attendance_history_provider.dart` (the delete flow)
   - Safe to change

6. **API contract**: Verify backend actually expects `phone` param. The user example shows `?phone=...&date=...` format.
