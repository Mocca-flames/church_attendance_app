# Plan: Add Delete Attendance via Long-Press Gesture

## Overview
Add ability to delete attendance records by long-pressing a card in the Attendance History screen. The delete operation will use the filtered DELETE endpoint (`contact_id` + `date`) to remove all attendance entries for that contact on that specific date.

## Assumptions
- The backend endpoint `DELETE /attendance/` already exists and accepts query parameters: `contact_id`, `date`, `date_from`, `date_to`, `service_type`.
- The typical use case is to delete by `contact_id` and a single `date` (current date in the history context).
- Existing single-record delete (`deleteAttendance(int id)`) remains unchanged; new method will be additive.
- Timezone handling: The backend converts dates to SAST; the client sends `date` as `YYYY-MM-DD` string or `date_from`/`date_to` as ISO8601.
- Offline behavior: Local deletion occurs immediately; remote deletion is attempted only if online (best-effort, failures are logged).

```dart
Future<int> deleteAttendancesByFilters({
  int? contactId,
  DateTime? date,
  DateTime? dateFrom,
  DateTime? dateTo,
  String? serviceType,
}) {
  final query = delete(attendances);
  
  var expr = const True();
  if (contactId != null) {
    expr = expr & attendances.contactId.equals(contactId);
  }
  if (date != null) {
    final start = DateTime(date.year, date.month, date.day);
    final end = start.add(const Duration(days: 1));
    expr = expr & attendances.serviceDate.isBiggerOrEqualValue(start) &
                attendances.serviceDate.isSmallerThanValue(end);
  } else {
    if (dateFrom != null) {
      expr = expr & attendances.serviceDate.isBiggerOrEqualValue(dateFrom);
    }
    if (dateTo != null) {
      final endOfDay = DateTime(dateTo.year, dateTo.month, dateTo.day, 23, 59, 59, 999);
      expr = expr & attendances.serviceDate.isSmallerOrEqualValue(endOfDay);
    }
  }
  if (serviceType != null) {
    expr = expr & attendances.serviceType.equals(serviceType);
  }
  
  query.where((t) => expr);
  return query.go();
}
```

```dart
Future<void> deleteAttendanceFiltered({
  int? contactId,
  DateTime? date,
  DateTime? dateFrom,
  DateTime? dateTo,
  ServiceType? serviceType,
}) async {
  try {
    final queryParams = <String, dynamic>{};
    
    if (contactId != null) {
      queryParams['contact_id'] = contactId;
    }
    if (date != null) {
      queryParams['date'] = '${date.year}-${date.month.toString().padLeft(2, '0')}-${date.day.toString().padLeft(2, '0')}';
    } else {
      if (dateFrom != null) {
        queryParams['date_from'] = dateFrom.toUtc().toIso8601String();
      }
      if (dateTo != null) {
        queryParams['date_to'] = dateTo.toUtc().toIso8601String();
      }
    }
    if (serviceType != null) {
      queryParams['service_type'] = serviceType.backendValue;
    }

    await _dioClient.dio.delete(
      ApiConstants.attendanceDelete,
      queryParameters: queryParams.isNotEmpty ? queryParams : null,
    );
  } on DioException catch (e) {
    if (e.type == DioExceptionType.connectionError ||
        e.type == DioExceptionType.unknown) {
      throw const AttendanceRemoteException(
        'Network error',
        isNetworkError: true,
      );
    }
    rethrow;
  }
}
```

```dart
@override
Future<void> deleteAttendanceFiltered({
  int? contactId,
  DateTime? date,
  DateTime? dateFrom,
  DateTime? dateTo,
  ServiceType? serviceType,
}) async {
  // Delete locally first (optimistic)
  await _localDataSource.deleteAttendanceFiltered(
    contactId: contactId,
    date: date,
    dateFrom: dateFrom,
    dateTo: dateTo,
    serviceType: serviceType?.backendValue,
  );

  // Attempt remote deletion if online
  if (await _isOnline()) {
    try {
      await _remoteDataSource.deleteAttendanceFiltered(
        contactId: contactId,
        date: date,
        dateFrom: dateFrom,
        dateTo: dateTo,
        serviceType: serviceType,
      );
    } on AttendanceRemoteException catch (e) {
      // Log but don't throw; local deletion already succeeded
      print('Remote delete filtered failed: ${e.message}');
    } catch (e) {
      print('Remote delete filtered error: $e');
    }
  }
}
```

```dart
Future<void> deleteAttendanceForContact({
  required int contactId,
  required DateTime date,
}) async {
  state = state.copyWith(clearError: true);
  try {
    await _repository.deleteAttendanceFiltered(
      contactId: contactId,
      date: date,
    );
    await loadAttendances(); // Refresh the displayed list
  } catch (e) {
    state = state.copyWith(error: e.toString());
  }
}
```

## Additional Considerations
- **Offline behavior**: Local deletion is immediate; remote deletion is attempted only if online. If remote fails (network/error), it is logged but not shown to user.
- **Error handling**: Repository exceptions are caught by the provider and stored in state; the screen's `ref.listen` displays a SnackBar automatically.
- **Date handling**: Only the date part of `serviceDate` is used for deletion; time component ignored.
- **Refresh**: After deletion, `loadAttendances()` reloads the list from local DB, reflecting changes.
- **Bulk delete**: Deleting by `contact_id` + `date` removes all service types for that contact on that date.
- **State**: No new state variables (e.g., `isDeleting`) are introduced to keep changes minimal; UI may briefly remain interactive during delete, which is acceptable for this operation.
