# Plan: Enhance Date Functionality for Attendance History

## Objective
Transform the date filtering from a date range (default: last 30 days) to a **single date mode** (default: today). Users must manually tap to switch to date range mode. This prepares the system for future features while maintaining backward compatibility with PDF export.

## Current State Analysis
- **State**: Uses `dateFrom` and `dateTo` as DateTime fields
- **Default**: Last 30 days range (`now.subtract(Duration(days: 30))` to `now`)
- **UI**: Date range picker with always-visible range selector
- **PDF Export**: Already supports both single date and range via `downloadAttendancePdf(date: ..., dateFrom: ..., dateTo: ...)`
- **Repository**: `getAttendanceHistory` accepts `dateFrom` and `dateTo` parameters

## Proposed Changes

### 1. Add DateFilterMode Enum
Create a new enum to distinguish between filtering modes:
```dart
enum DateFilterMode {
  singleDate, // Single day (default)
  dateRange,  // Custom date range
}
```

### 2. Update AttendanceHistoryState
- Add `DateFilterMode mode` field
- Keep `dateFrom` and `dateTo` but reinterpret them:
  - When `mode == singleDate`: `dateFrom == dateTo` represents the selected day
  - When `mode == dateRange`: `dateFrom` and `dateTo` represent the range
- Default state: `mode = DateFilterMode.singleDate`, both dates = today (DateTime.now() truncated to day)

### 3. Update AttendanceHistoryNotifier
- **build()**: Initialize with `mode = DateFilterMode.singleDate`, `dateFrom = dateTo = DateTime.now()` (midnight)
- **setDateRange(dateFrom, dateTo)**: Set mode to `DateFilterMode.dateRange` and update dates
- **setSingleDate(date)**: New method - set mode to `DateFilterMode.singleDate` and set both dates to the selected day (midnight)
- **toggleMode()**: Optional helper to switch between modes
- **loadAttendances()**: Pass dates as-is to repository (already works)
- **downloadPdf()**: Already handles single vs range detection via `_isSameDay()` - keep this logic

### 6. Edge Cases & Considerations

- **Initial load**: Ensure provider loads attendances with correct single-day query

- **Backward compatibility**: Existing code using dateFrom/dateTo continues to work

## Trade-offs Considered
- **Single field vs two fields**: Keeping two DateTime fields avoids major API changes; repository already accepts nullable dateFrom/dateTo which works fine for single-day queries
- **Mode toggle visibility**: Toggle will be subtle to avoid clutter
- **Default behavior change**: This is a breaking change from 30-day range to today only - acceptable per user request

## Future Extensibility
- The `DateFilterMode` field can accommodate additional modes later (e.g., "last 7 days", "this week", "custom range")
- The UI structure supports easy addition of mode-selection options
